# Artifact-Card Failure-Modes Contrast v1 Scaffold

Date: 2026-04-30

## Goal

Create the next narrow supervision branch after the broad evidence-conditioned branch failed downstream.

This branch keeps the task focused on `primary_failure_modes`, but it no longer asks the model to score all 8 labels at once.

Instead, it asks for exactly one local contrast-group decision in each of the four highest-value confusion boundaries around `missing-required-detail`.

## Why this branch exists

The current decomposition sequence says three important things:

- `artifact-card-failure-modes-pairwise-v1` is still the strongest downstream branch so far because it reached reconstructed held-out top-2 set match `0.25`.
- `artifact-card-failure-modes-evidence-v1` improved row-level structure and local fields, but reconstructed top-2 set match fell back to `0.0`.
- The broad evidence branch changed the failure mode from dominant-label collapse into broad over-acceptance.

So the next move is not another full 8-label rewrite.

The next move is to narrow the supervision to the exact contrast groups where `missing-required-detail` kept swallowing nearby labels.

## Contrast groups

This branch uses exactly four local decision groups:

- `missing-required-detail` vs `generic-explanation`
- `missing-required-detail` vs `no-material-change`
- `missing-required-detail` vs `hallucinated-detail`
- `missing-required-detail` vs `overlap-contaminated-eval`

## Branch shape

Source dataset:
- `data/artifact-card-failure-modes-v1/`

Generated dataset:
- `data/artifact-card-failure-modes-contrast-v1/`

Builder:
- `scripts/build_failure_mode_contrast_dataset.py`

Evaluator:
- `scripts/evaluate_failure_mode_contrast_run.py`

Each original source example becomes 4 rows, one per contrast group.

Each row returns one strict JSON object with exactly:
- `contrast_group`
- `decision`
- `evidence_key`

Allowed `decision` values are local to the group:
- anchor label name, for example `missing-required-detail`
- rival label name, for example `generic-explanation`
- `both`
- `neither`

Interpretation:
- anchor label name = anchor supported, rival not supported
- rival label name = rival supported, anchor not supported
- `both` = both labels in the contrast group are supported
- `neither` = another label outside the contrast group is the better explanation

This gives exactly one local decision per contrast group while still allowing the branch to represent cases where both labels in the pair are genuinely present.

## Reconstruction rule

This branch only tries to reconstruct source examples whose gold 2-label set stays inside the 5-label contrast universe:

- `missing-required-detail`
- `generic-explanation`
- `no-material-change`
- `hallucinated-detail`
- `overlap-contaminated-eval`

For each reconstructable source example:
- each contrast-group row contributes 1 point to the chosen label
- `both` contributes 1 point to both labels in that contrast group
- `neither` contributes no points
- the reconstructed positive set is every label with score > 0
- the reconstructed top-2 set is the two labels with the highest scores

This is intentionally narrower than the broad evidence-conditioned branch.

## How to judge the first contrast-v1 run

First check row-level behavior:
- valid JSON rate
- exact row match
- `decision` accuracy
- `evidence_key` accuracy
- per-group decision confusion

Then check reconstruction on the 4 reconstructable held-out source examples:
- exact positive-label set match
- top-2 set match
- predicted positive-count histogram

Main comparison target:
- beat the evidence-v1 downstream failure (`0.0` reconstructed top-2 set match)
- try to beat or at least match pairwise-v1 on the downstream objective while using a much smaller and sharper supervision space
